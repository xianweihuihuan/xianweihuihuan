## About Me

我是一个偏爱底层、系统与长期主义的开发者。

比起把功能快速堆出来，我更在意把问题真正做深:  
理解它为什么成立，系统如何运转，瓶颈出现在哪里，优化是否真的有效。

技术对我的吸引力，不只在于 `can build`，更在于 `can understand`、`can implement`、`can explain`。

<br/>

## What I Value

我更看重能力结构是否在持续变得完整，而不是表面的项目数量。

我希望自己逐步拥有这样的能力:

- 对底层系统有足够扎实的理解
- 对性能问题有足够敏感的判断
- 对复杂系统有拆解、建模与分析能力
- 能把传统系统能力自然延伸到 AI Infrastructure

<br/>

## Engineering Style

我喜欢清晰、克制、可验证的工程表达。

- Less noise, more substance
- Less decoration, more depth
- Less slogan, more understanding
- Less surface, more structure
