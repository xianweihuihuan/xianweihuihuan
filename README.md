## Technical Summary

我现在的技术主线，大致可以概括成下面几层:

- `Systems`: `Linux` `Operating Systems` `System Programming` `Computer Networking`
- `Core Topics`: `Database Internals` `Algorithms` `Concurrency` `Performance Analysis`
- `Languages`: `C` `C++` `Go` `Python`
- `AI Infra`: `LLM Inference` `vLLM` `KV Cache` `Agents`

这条路线对我来说不是简单地切换方向，而是很自然的延伸:  
先把底层能力打牢，再把对系统、性能和复杂结构的理解，继续带到 AI 基础设施里。

<br/>

## What Matters To Me

我最在意的，不是做了多少项目，而是有没有逐渐形成一种稳定的技术判断力。

我希望自己慢慢长成这样的人:

- 能从原理层理解系统
- 能把关键机制真正实现出来
- 能对复杂问题做拆解、建模与分析
- 能用数据和证据，而不是感觉，来判断一个优化是否成立

<br/>

## Engineering Temperament

我喜欢的工程表达，是克制的，也是有分量的。

- 不追求表面的热闹，更重视结构上的完整
- 不满足于功能可用，更在意机制是否清楚
- 不迷恋快速堆砌，更相信长期积累后的稳定增长
